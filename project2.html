Catalyst Case Study

Designing a learning product from the ground up

The Problem

There are limited options for learning products designed exclusively for STEM topics. Those that exist lack full scale features users are looking for. Users are having trouble:

  1. 1. Searching for personalized science courses and content
  2. 2. Tracking progress towards course and learning goals
  3. 3. Staying motivated to finish self-paced courses

The Solution

Catalyst is a self-paced learning product that makes STEM education both fun and engaging. It offers specialized recommendations, tracks learning goals, and rewards users for completing learning objectives. Let's take a look:

My Roles

I was solely responsible for the research, user experience, user interface, and branding elements of this product. I collaborated weekly with a senior design mentor who provided guidance and feedback during each design phase.

Competitive Analysis

I took a closer look at three learning products that are currently on the market. I wanted to analyze the behaviors of competitors and assess different approaches to learning products.

Strengths

Duolingo and Lumosity are the most comparable to the concept of Catalyst. They both have strong branding and fun UI elements that are interesting to users. Both also incorporate course tracking and variable reward systems to keep users motivated. YouTube provides smart recommendations to users based on interactions within the product.

Weaknesses

Competitors only offer vague course descriptions and lack metrics (like reviews) that are relevant to users while content browsing. After enrolling, users can only progress linearly and there are no options to personalize content based on specific user interests.

The Key Takeaways

I used the data collected from the survey and competitive analysis to outline some actionable insights. Catalyst will find success by incorporating:

  1. 1. Fun, engaging visuals without sacrificing functionality
  2. 2. Detailed but concise course overviews and progress tracking
  3. 3. Extrinsic motivators for improved user engagement and retention

User Flows

I created user flows to represent how a user would interact with TheraSelf to accomplish the high priority user stories. From there, a site map was created to help organize the destinations for each of the user flows.

Sketches

I made rapid iteration sketches to explore different configurations for my product pages. I was able to mitigate potentional breakdowns in user flows and gained a clearer direction for the overall function and design of the product.

Wireframes

Using my user flows and sketches as points of reference, I created wireframes in Figma. Wireframes were compiled to represent how users can accomplish the most important product tasks. I was able to identify some inconsistencies and breakdowns in userflow that will be remedied in my mockups.

Adding a Course

Users browse content from a course feed on the home page. A course description and lesson overview display when a specific course is selected. After adding a course, users return to the home page where they are able to browse for additional content or select the learn tab from the toolbar to begin course content.

Accessing Lesson Content

Each course has a content page where individual lessons are selected. Users review the lesson content and key learning points before navigating to quizzes or interactive games for knowledge checks.

Mockups

At this point, I breathed life into my wireframes by applying the color palette, illustrations, and other visual elements. Overall functionality was improved upon as well through. Let's take a look at the evolution of Catalyst across the different iterations.

Homepage Configuration

There was not much logic to the grid display of course content in the wireframe. The second iteration sees the addition of category filters and places greater emphasis on personalized recommendations. Both of these features save the user time and effort.

Lesson Overview

The second iteration of the lesson overview page now mirrors the configuration of the course overview for improved consistency. It uses expandble accordian tabs to house comprehensive progress tracking information. I was also able to incorporate collaborative features, like forums, by using space more effectively above the lesson breakdown.

Prototyping & Testing

With the preliminary revisions in place, I went on to create a clickable, interactive prototype. I then conducted remote usability testing with a handful of users. Their feedback shaped the subsequent modifications to Catalyst.

Poor feedback during quizzes

Users reported a lack of feedback while interacting with quiz pages. Specifically, there was no feedback when selecting answers to quiz questions. This was confusing to users and caused an unintended breakdown in the flow between lesson content and knowledge checks. These frustrations could pose an issue with user retention long term.

Iteration 2

As users proceed to the lesson quiz, the submission button remains in an inactive state until users select a response. The background color of the selected response changes to a light green. This is a simple and effective method to improve navigation feedback without distracting from the content itself.

Concluding Thoughts

I kept the scope of the Catalyst prototype relatively narrow for purposes of this case study. There's room to include additional screens (like the updated dashboard with category filters applied) to improve the overall functionality of the product. Applying animations throughout the product would elevate Catalyst to the next level as well.
